Mod logs record server events and moderation actions in a channel you designate. Each event posts a color-coded embed with relevant details, giving your moderation team a clear audit trail.

## Configure mod logs

<Steps>
  <Step title="Open the dashboard">
    Go to the [BrewBot Dashboard](https://brewbot.techbrew.dev) and select your server.
  </Step>
  <Step title="Navigate to Mod Logs">
    Open the **Mod Logs** page from the sidebar.
  </Step>
  <Step title="Select a log channel">
    Choose the channel where the bot should post log entries. You can also create a new `#mod-logs` channel directly from the dashboard.
  </Step>
  <Step title="Enable logging">
    Turn on the **Enable Audit Logging** toggle.
  </Step>
  <Step title="Choose events to log">
    Enable the events you want to track. Events are organized into seven categories — see the full reference below.
  </Step>
</Steps>

<Tip>
  Enable only the events you care about. Logging too many events can make the log channel noisy and harder to review.
</Tip>

## Additional settings

- **Ignore bots** — don't log actions made by bots (on by default)
- **Hide user thumbnails** — don't display user avatars in log embeds
- **Ignored channels** — skip message delete/edit logging for specific channels

## Event reference

Each log entry is a Discord embed with a color indicating the action type:

- 🟢 **Green** — joins, creates, adds
- 🟡 **Yellow** — updates, changes, edits
- 🔴 **Red** — deletes, bans, leaves
- 🔵 **Blue** — informational
- 🟠 **Orange** — mutes, warnings

### Member events

<Tabs>
  <Tab title="Member joined">
    Logs when a new member joins the server.

    | Field | Value |
    | --- | --- |
    | **Color** | Green |
    | **Description** | Member mention, **joined the server** |
    | **Details** | Account creation date, current member count |
    | **Footer** | User ID |
  </Tab>
  <Tab title="Member left">
    Logs when a member leaves or is kicked from the server.

    | Field | Value |
    | --- | --- |
    | **Color** | Red |
    | **Description** | Member mention, **left the server** |
    | **Details** | Roles the member had, current member count |
    | **Footer** | User ID |
  </Tab>
  <Tab title="Member banned">
    Logs when a member is banned.

    | Field | Value |
    | --- | --- |
    | **Color** | Red |
    | **Description** | Member mention, **was banned** |
    | **Details** | Ban reason (or "No reason provided") |
    | **Footer** | User ID |
  </Tab>
  <Tab title="Member unbanned">
    Logs when a ban is removed from a member.

    | Field | Value |
    | --- | --- |
    | **Color** | Green |
    | **Description** | Member mention, **was unbanned** |
    | **Footer** | User ID |
  </Tab>
  <Tab title="Nickname changed">
    Logs when a member's nickname changes.

    | Field | Value |
    | --- | --- |
    | **Color** | Yellow |
    | **Description** | Member mention, **nickname changed** |
    | **Details** | Before and after nicknames |
    | **Footer** | User ID |
  </Tab>
  <Tab title="Roles changed">
    Logs when a member's roles are added or removed.

    | Field | Value |
    | --- | --- |
    | **Color** | Yellow |
    | **Description** | Member mention, **roles have changed** |
    | **Details** | Added roles, removed roles |
    | **Footer** | User ID |
  </Tab>
</Tabs>

### Message events

<Tabs>
  <Tab title="Message deleted">
    Logs when a message is deleted. Skipped for ignored channels.

    | Field | Value |
    | --- | --- |
    | **Color** | Red |
    | **Description** | Message author, **deleted in** channel mention |
    | **Details** | Message content (truncated to 1,000 characters) |
    | **Footer** | Message ID |
  </Tab>
  <Tab title="Message updated">
    Logs when a message is edited. Skipped for ignored channels.

    | Field | Value |
    | --- | --- |
    | **Color** | Yellow |
    | **Description** | Message author, **edited in** channel mention |
    | **Details** | Before and after content (each truncated to 400 characters), link to message |
    | **Footer** | Message ID |
  </Tab>
  <Tab title="Invite posted">
    Logs when a server invite is created.

    | Field | Value |
    | --- | --- |
    | **Color** | Blue |
    | **Title** | Invite Created |
    | **Details** | Invite code, channel, created by, max uses (or "Unlimited"), expiry (or "Never") |
  </Tab>
</Tabs>

### Role events

<Tabs>
  <Tab title="Role created">
    Logs when a new role is created.

    | Field | Value |
    | --- | --- |
    | **Color** | The role's color (green fallback) |
    | **Title** | Role Created |
    | **Details** | Role name, color hex, mentionable (yes/no), hoisted (yes/no) |
    | **Footer** | Role ID |
  </Tab>
  <Tab title="Role updated">
    Logs when a role's properties change. Only changed properties are shown.

    | Field | Value |
    | --- | --- |
    | **Color** | The role's new color (yellow fallback) |
    | **Title** | Role Updated |
    | **Details** | Changed properties: name, color, hoisted, mentionable (before → after) |
    | **Footer** | Role ID |
  </Tab>
  <Tab title="Role deleted">
    Logs when a role is deleted.

    | Field | Value |
    | --- | --- |
    | **Color** | Red |
    | **Title** | Role Deleted |
    | **Details** | Role name, color hex |
    | **Footer** | Role ID |
  </Tab>
</Tabs>

## Default settings

Moderation events (member muted, unmuted, moderation ban, moderation unban) are enabled by default. All other events start disabled. You can toggle any event on or off at any time.
